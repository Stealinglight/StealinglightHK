## Spacing Scale

Declared values (multiples of 4, all in use across existing components):

| Token | Value     | Usage                                                                                             |
| ----- | --------- | ------------------------------------------------------------------------------------------------- |
| xs    | 4px (1)   | Icon gaps, inline padding, credential dot size (w-1.5)                                            |
| sm    | 8px (2)   | Compact element gaps, pill button inline padding, filter pill gap                                 |
| md    | 16px (4)  | Default element spacing, section horizontal padding (px-4), card padding                          |
| lg    | 24px (6)  | Section vertical padding base (py-24), grid gap (gap-6), card info padding (p-6)                  |
| xl    | 32px (8)  | Section vertical padding desktop (md:py-32), section heading margin-bottom (mb-16 = 64px at 4rem) |
| 2xl   | 48px (12) | Major section breaks, hero CTA bottom margin (mb-12)                                              |
| 3xl   | 64px (16) | Page-level section breaks, hero content bottom margin area                                        |

Exceptions:

- **Touch targets:** Filter pill buttons minimum 44px hit area (11 in Tailwind) for WCAG compliance on mobile
- **Scroll progress bar:** 2px height (h-0.5) -- thinner than scale minimum, intentional for subtlety
- **Preloader logo text:** Vertically centered in viewport, no fixed spacing token -- uses flexbox centering

**Source:** Existing section patterns in Hero.tsx, Portfolio.tsx, About.tsx, Services.tsx, Contact.tsx. Standard `py-24 md:py-32` and `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8` container patterns from CLAUDE.md.

## Animation Contract

### Easing

| Name                     | Value                                               | Usage                                                                                                 |
| ------------------------ | --------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| Cinematic                | `[0.16, 1, 0.3, 1]`                                 | All phase 3 animations (hero reveal, scroll triggers, filter transitions). Smooth deceleration curve. |
| Spring (scroll progress) | `{ stiffness: 100, damping: 30, restDelta: 0.001 }` | Scroll progress bar only -- useSpring for smooth tracking                                             |

**Source:** D-01 locked decision.

### Hero Reveal Timeline (D-02)

| Element                           | Delay | Duration | Animation                         | Variants Key   |
| --------------------------------- | ----- | -------- | --------------------------------- | -------------- |
| Name (CHRIS MCMILLON)             | 0.3s  | 1.5s     | tracking 0.5em to 0.3em + fade-in | `trackingFade` |
| Headline (Cinematographer)        | 0.6s  | 0.8s     | fade-up 30px + blur 8px to 0px    | `fadeUpBlur`   |
| Subtitle (Aerial photographer...) | 0.9s  | 0.8s     | fade-up 30px + blur 8px to 0px    | `fadeUpBlur`   |
| Tagline                           | 1.2s  | 1.0s     | fade-in only                      | `fadeIn`       |
| CTAs                              | 1.6s  | 0.8s     | fade-up 20px                      | `fadeUp`       |
| Scroll indicator                  | 2.0s  | 1.0s     | fade-in                           | `fadeIn`       |

Implementation note: Use per-element explicit `delay` values via the `custom` prop pattern (not `staggerChildren`) because the timeline has non-uniform spacing (0.3s gaps then 0.4s gaps). See RESEARCH.md Open Question 1.

### Filter Grid Transition (D-04)

| Phase         | Duration | Animation                                  |
| ------------- | -------- | ------------------------------------------ |
| Exit          | 0.2s     | opacity 1 to 0 + scale 1 to 0.9            |
| Layout reflow | 0.3s     | Motion `layout` prop with cinematic easing |
| Enter         | 0.3s     | opacity 0 to 1 + scale 0.9 to 1            |

Use `AnimatePresence mode="popLayout"` with `layout` prop on each card. Grid container must have `position: relative`.

### Preloader (D-06, D-07, D-08)

| Element                    | Animation                        | Duration       |
| -------------------------- | -------------------------------- | -------------- |
| Brand text (STEALINGLIGHT) | Amber opacity pulse 0.5 to 1.0   | 1.5s infinite  |
| Progress bar               | Width 0% to 100% (synthetic)     | ~0.8s ease-out |
| Dismiss                    | Opacity 1 to 0 + scale 1 to 1.02 | 0.5s           |

Dismiss condition: `heroVideoCanPlay AND minTimePassed(800ms)`. Safety timeout: 4s maximum.

### Scroll-Triggered Section Animations (VISL-02, Claude's Discretion)

| Section                  | Animation       | Duration | Stagger       |
| ------------------------ | --------------- | -------- | ------------- |
| Portfolio heading        | fade-up 20px    | 0.8s     | none          |
| Portfolio featured video | fade-up 40px    | 0.8s     | none          |
| Portfolio grid cards     | fade-up 40px    | 0.6s     | 0.1s per card |
| About image              | fade-left 40px  | 0.8s     | none          |
| About text               | fade-right 40px | 0.8s     | none          |
| Services heading         | fade-up 20px    | 0.8s     | none          |
| Services cards           | fade-up 30px    | 0.6s     | 0.1s per card |
| Contact heading          | fade-up 20px    | 0.8s     | none          |
| Contact info cards       | fade-up 30px    | 0.8s     | 0.2s delay    |
| Contact form             | fade-up 30px    | 0.8s     | 0.4s delay    |
| Footer                   | fade-up 20px    | 0.6s     | none          |
| Clients heading          | fade-up 20px    | 0.6s     | none          |
| Clients marquee          | fade-in         | 0.8s     | 0.2s delay    |

All use `viewport={{ once: true }}` and cinematic easing `[0.16, 1, 0.3, 1]`. Existing patterns preserved; easing curves upgraded from current `[0.25, 0.1, 0.25, 1]`.

### Marquee (VISL-03, Claude's Discretion)

| Property     | Value                                                                 |
| ------------ | --------------------------------------------------------------------- |
| Direction    | left-to-right continuous scroll                                       |
| Speed        | 30s per full cycle (adjust visually)                                  |
| Pause        | on hover via `animation-play-state: paused`                           |
| Logo count   | All 15 available logos (duplicated for seamless loop = 30 rendered)   |
| Logo height  | 32px (h-8) mobile, 40px (h-10) desktop                                |
| Logo gap     | 48px (gap-12)                                                         |
| Logo opacity | 30% default, 60% on hover, white filter via `brightness(0) invert(1)` |
| Duplicates   | `aria-hidden="true"` on second set for screen readers                 |
| CSS keyframe | `translateX(0)` to `translateX(calc(-50% - 1.5rem))`                  |

### Touch Interaction (D-09, D-10)

| State                  | Trigger               | Visual                                                                |
| ---------------------- | --------------------- | --------------------------------------------------------------------- |
| Default (touch device) | Page load             | Thumbnail with gradient overlay, card info visible                    |
| Tapped once            | First tap on card     | Video preview plays in-place, "TAP TO WATCH" overlay appears centered |
| Modal open             | Second tap            | Full video modal opens                                                |
| Deselected             | Tap elsewhere on page | Preview stops, returns to default state                               |

Detection: CSS `@media (hover: hover)` for hover-capable devices. React state `tappedId` manages which card is in preview mode on touch devices. Use `onClick` (not `onPointerDown`) to avoid scroll-tap conflicts per RESEARCH.md Pitfall 5.

"TAP TO WATCH" overlay styling: 14px, weight 500, tracking 0.05em, uppercase, white text on semi-transparent black pill (`bg-cinematic-black/70 px-4 py-2 rounded-full`).

### Modal Keyboard Navigation (VIDO-04, Claude's Discretion)

| Key        | Action                                                                              |
| ---------- | ----------------------------------------------------------------------------------- |
| Escape     | Close modal, restore body scroll, return focus to triggering card                   |
| ArrowLeft  | Navigate to previous video in current filtered list                                 |
| ArrowRight | Navigate to next video in current filtered list                                     |
| Space      | Toggle play/pause on current video (with `e.preventDefault()` to block page scroll) |

Focus management: Modal container receives focus on open (`tabIndex={-1}`, `ref.focus()`). Body overflow set to `hidden` while modal is open.

### Scroll Progress Indicator (QUAL-01, Claude's Discretion)

| Property         | Value                                                 |
| ---------------- | ----------------------------------------------------- |
| Position         | fixed, top: 0, full width                             |
| Height           | 2px (h-0.5)                                           |
| Color            | `bg-cinematic-amber`                                  |
| Z-index          | z-[60] (above nav z-50, below preloader z-[70])       |
| Transform origin | left                                                  |
| Tracking         | Motion `useScroll()` + `useSpring()` driving `scaleX` |

### Reduced Motion

When `prefers-reduced-motion: reduce` is active (detected via Motion's `useReducedMotion()` hook):

- All fade-up/blur/tracking animations reduce to opacity-only with 0.01s duration
- Marquee animation paused (CSS `prefers-reduced-motion` media query)
- Filter grid transitions reduce to cross-fade only (no scale)
- Preloader dismisses with simple opacity fade
- Scroll progress bar still tracks (no motion, just position)

**Source:** D-01 through D-10 locked decisions, RESEARCH.md architecture patterns, RESEARCH.md reduced motion section.
