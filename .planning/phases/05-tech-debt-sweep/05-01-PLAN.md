**SectionErrorBoundary.tsx** — The `handleSectionError` function on line 11 has parameter `error: Error` but `react-error-boundary`'s `onError` expects `error: unknown`. Also, `info` parameter uses `{ componentStack?: string | null }` but the library expects `ErrorInfo` from React.

Change line 11 from:

```typescript
function handleSectionError(error: Error, info: { componentStack?: string | null }) {
```

to:

```typescript
function handleSectionError(error: unknown, info: { componentStack?: string | null | undefined }) {
```

Update the console.error call to handle `unknown` error type. Change:

```typescript
console.error('[SectionError]', error.message, info.componentStack);
```

to:

```typescript
console.error(
  '[SectionError]',
  error instanceof Error ? error.message : String(error),
  info.componentStack
);
```

**videos.ts** — The `videoProjects` array uses `as const` which makes each element a distinct literal type. Only item id:1 has `featured: true`, but `featuredVideo` on line 188 and `gridVideos` on line 191 filter on `.featured` which does not exist on most element types.

Fix: Add an explicit `VideoProject` interface and type the array with it instead of using `as const`. Define:

```typescript
interface VideoProject {

  title: string;
  category: string;
  description: string;
  duration: string;
  videoUrl: string;
  posterUrl: string;
  featured?: boolean;
}
```

Change line 6 from `export const videoProjects = [` to `export const videoProjects: VideoProject[] = [`.

Remove the `as const` assertion on line 179.

Keep the `heroVideo` object and derived exports (`featuredVideo`, `gridVideos`) unchanged — they will now type-check because `VideoProject` has `featured?: boolean`.

Export the `VideoProject` type: `export type { VideoProject };` at the bottom of the file.
